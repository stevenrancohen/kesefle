🔒 Encryption

הצפנה — בלי קיצורי דרך

כל סוד אצלנו מוצפן עם אלגוריתמים מאושרי NIST, באמצעות ספריית node:crypto מבוססת OpenSSL — בלי תלויות npm חיצוניות שיכולות להידלף.

🛡️

במנוחה (at rest)

AES-256-GCM עם IV אקראי של 12 בתים ותג אימות של 16 בתים. כל refresh token של Google נשמר ב-KV רק כ-envelope מוצפן. 

v1:<kid>:<iv>:<tag>:<ciphertext>
🚄

במעבר (in transit)

TLS 1.3 בכל endpoint, HSTS עם max-age=63072000 (שנתיים) ו-preload-ready. אין HTTP חשוף, נקודה.

A+ on SSL Labs
✍️

אימות חתימות

HMAC-SHA256 עם timingSafeEqual על webhooks של WhatsApp ותשלומים. RS256 עם JWKS של Google ותמיכת kid rotation.

lib/crypto.js צפה במקור ב-GitHub → 
// AES-256-GCM, envelope carries kid for KEK rotation.
// We fail closed: no key configured -> encrypt() throws.
export function encrypt(plaintext, aad) {
  const { key, kid } = loadActiveKey();
  const iv  = crypto.randomBytes(12);
  const cph = crypto.createCipheriv('aes-256-gcm', key, iv);
  if (aad) cph.setAAD(Buffer.from(aad));
  const ct  = Buffer.concat([cph.update(plaintext, 'utf8'), cph.final()]);
  return `v1:${kid}:${b64u(iv)}:${b64u(cph.getAuthTag())}:${b64u(ct)}`;
}
💡

מה זה אומר בשבילך

גם אם מישהו יצליח להעתיק את כל מסד הנתונים שלנו (KV), הוא יקבל מחרוזות בייטים חסרות משמעות. כדי לחלץ אפילו refresh token אחד, הוא יצטרך גם את ה-KEK שמאוחסן ב-Vercel Secrets בנפרד. הצפנה לבד לא מספיקה — אבל כשלא נופלת על נקודה אחת, היא דמוקרטית כלפי מי שמנסה לתקוף אותך.

🗄️ Data sovereignty

מי שומר מה

הנתונים שלך לא חיים בענן שלנו. הם חיים בענן שלך. אנחנו רק מתחברים — והקשר ניתן לניתוק בלחיצה אחת.

מה ה-scope drive.file מרשה?

רק לקבצים שכספ'לה יצרה. לא יכולים לקרוא, לעדכן או למחוק שום קובץ אחר ב-Drive שלך — אפילו לא לראות שהוא קיים. זה ה-scope הכי מצומצם ש-Google מציעה לעבודה עם Sheets.

איפה הם השרתים?

כרגע אזור Vercel US-East (Washington DC). מעבר ל-Frankfurt EU-Central בתהליך, בהתאם לדרישות תיקון 13. ההתקדמות פתוחה ב-דף הסטטוס.

🛡️ Hardening

הקשחה — defense in depth

אבטחה זו לא קיר אחד גבוה, אלא הרבה גדרות נמוכות. הנה מה שמופעל בכל בקשה:

  • HSTS — preload-ready
    Strict-Transport-Security: max-age=63072000; includeSubDomains; preload
  • CSP — Content-Security-Policy

    מקורות סקריפטים נעולים. ללא unsafe-eval. דיווח חוזר ב-report-to.

  • X-Frame-Options: DENY

    אף אתר לא יכול לטעון אותנו ב-iframe — אפס הגנה לחטיפת קליקים.

  • Permissions-Policy

    חוסם מצלמה, מיקרופון, מיקום, חיישנים — כל מה שאין לנו צורך בו.

  • Rate limiting

    Token bucket לכל endpoint רגיש. 10 נסיונות התחברות ב-5 דקות, 2 ייצוא לשעה.

  • Formula-injection sanitization

    כל ערך לפני כתיבה לגיליון מנוטרל מ-=, +, -, @ פותחים.

  • Webhook signatures

    WhatsApp + תשלומים — HMAC על raw body, השוואה ב-timingSafeEqual.

  • JWT verification

    RS256 מול JWKS של Google. תמיכה ב-kid rotation וקאש של 60 שניות.

  • Server-side input validation

    סכמת JSON לכל בקשה. דחייה ב-400 לפני שמגיעים לדאטה.

  • Audit log

    כל פעולת admin (מחיקה, ייצוא, החלפת תוכנית) נכתבת ל-append-only log עם reqId.

❓ Trust FAQ

שאלות נפוצות

השאלות שלקוחות עסקיים שואלים אותנו — והתשובות הכנות.

מה קורה אם תיסגרו?

הגיליון שלך נשאר ב-Google Drive שלך — בבעלות מלאה, בלי קשר אלינו. אנחנו פשוט נעלמים. כל פונקציות הסיכום והגרפים שב-Sheets ימשיכו לעבוד, כי הם מבוססים על נוסחאות מקומיות. במקרה של סגירה מוסדרת, נשלח 90 יום מראש הודעה + סקריפט ייצוא חד-פעמי שמייבא הכל לקובץ .xlsx.

האם תוכלו לראות את הנתונים שלי?

לא. הקוד שכותב לגיליון רץ ב-serverless ולא חוזר אלינו עם תוכן הגיליון. החריג היחיד: אם פתחת קריאת תמיכה ואישרת מפורשות בכתב — מפתח אחד מורשה יכול לבקש הרשאה זמנית של 24 שעות, שמוגבלת לקובץ ספציפי, ונרשמת באודיט לוג. אחרת — אף אחד אצלנו לא מסתכל.

איפה השרתים שלכם?

כרגע ב-Vercel US-East (Washington DC) + Upstash KV באותו אזור. אנחנו במהלך מעבר ל-EU-Central (Frankfurt) כדי להתאים לדרישות תיקון 13 בנושא העברת נתונים בינלאומית. ההתקדמות פתוחה ב-דף הסטטוס.

מה אם תיפרץ הצפנה?

Defense in depth: (1) האלגוריתם AES-256-GCM הוא NIST-approved ו-FIPS-compliant — שבירה מעשית דורשת מחשב קוונטי שעדיין לא קיים. (2) ה-AAD קושר כל ciphertext ל-userSub — לא ניתן להחליף envelope-ים בין משתמשים. (3) ה-KEK רוטר אחת לרבעון, עם תמיכה ב-kid rotation שמאפשרת רוטציה בלי downtime. (4) אם בכל זאת — Incident Response נכנס לפעולה תוך 5 דקות, ו-72 שעות הודעת PPA.

האם תוכלו למחוק את הגיליון בגוגל דרייב?

לא. ה-scope drive.file שביקשנו מ-Google מאפשר רק עבודה עם קבצים שאנחנו יוצרים. אנחנו לא יכולים לראות, לעדכן או למחוק שום קובץ אחר ב-Drive שלך. אפילו את הגיליון שיצרנו — לא נמחק אוטומטית בעת מחיקת חשבון; הוא נשאר אצלך עד שתחליט.

האם נכון שלכל חשבון יש refresh token שניתן לגנוב?

כן — זה הקומפרומיס של OAuth. אבל: (א) ב-KV הוא מאוחסן רק מוצפן AES-256-GCM, (ב) הצפנה קשורה ל-AAD ב-userSub כדי שמהפכי envelope-ים לא יעזרו, (ג) הוא רוטר אוטומטית בכל 90 יום, (ד) אתה יכול לבטל אותו ב-1 קליק ב-myaccount.google.com/permissions. גם אם תוקף ישיג עותק של ה-KV ויצליח לפצח את ההצפנה, יש לך כפתור עצירה מיידי שלא תלוי בנו.

מי רואה את ההודעות שאני שולח לבוט?

Meta מעבירה את ההודעה אלינו דרך WhatsApp Business API (חתומה ב-HMAC). אנחנו מפעילים עליה את הפרסר, שמחלץ סכום וקטגוריה, ושומרים רק את התוצאה המעובדת בגיליון שלך. הטקסט הגולמי נשמר ב-log ל-עד 90 ימים לצורך תמיכה ושיפור הפרסר, ואז נמחק אוטומטית. הוא לא חוזר ל-Meta, לא ל-Google, ולא ל-AI חיצוני.

איך אני יודע שאתם באמת עושים את כל זה?

הקוד פתוח לעיון ב-GitHub. ה-headers שלנו ניתנים לבדיקה ב-securityheaders.com. ה-SSL ב-SSL Labs. דוחות ה-Red Team כפי שצוין למעלה. אם משהו פה לא תואם — דווח לנו ויש מסלול bug bounty.
