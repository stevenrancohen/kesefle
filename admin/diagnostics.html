בודק תגובות בוט

בנה payload סימולציה של webhook מ-Meta. אינו שולח הודעה — רק מציג מה היה מגיע.

ה-stub אינו מפעיל את הבוט בפועל — רק מציג את ה-payload. 

    

  


  
  

    

      
        ›
        מה אומרת כל בדיקה?
      
      

        

          
1. בריאות Vercel — /api/health

          
בודק שה-API חי, ש-KV (מסד הנתונים) מגיב, ושמשתני הסביבה החיוניים מוגדרים. אם נופל — האתר עצמו לא יעבוד.

        

        

          
2. מספר הבוט (Meta WhatsApp)

          
בודק ש-META_PHONE_NUMBER_ID ו-META_ACCESS_TOKEN מוגדרים. אם לא — הבוט לא יכול לשלוח תגובות בחזרה.

        

        

          
3. כתובת webhook ב-Meta

          
מציג את ה-URL שצריך להיות מוגדר ב-Meta Developer Console. אימות בפועל דורש Graph API token — להריץ ידנית.

        

        

          
4. Apps Script webhook

          
אם הבוט הישן (Google Apps Script) עדיין פעיל, יש URL שצריך לבדוק בדפדפן/curl. הבדיקה כאן רק מציגה את ה-URL.

        

        

          
5. Anthropic API (Claude)

          
בודק שיש ANTHROPIC_API_KEY מוגדר. בלעדיו הבוט עובד אבל בלי AI — fallback לכללי regex.

        

        

          
6. אימות Google OAuth

          
בדיקה ידנית: בקונסולת Google Cloud — ספירת test users מאושרים והאם האפליקציה ב-Production או Testing.

        

        

          
7. בודק תגובות בוט (stub)

          
בונה payload בפורמט שמטא היתה שולחת, אבל אינו מפעיל את הבוט. שמש לאימות שהתשתית מקבלת payloads תקינים.

        

        

          
8. DNS / SSL

          
בודק שאפשר להגיע ל-kesefle.vercel.app ושה-HTTPS תקין. אם נופל — או DNS שבור או Vercel down.

        

        

          
9. מונה שגיאות 24 שעות

          
מציג את ערך errors:24h ב-KV. ערך 0 או null פירושו אין שגיאות שנאספו אחרון — או שאיסוף שגיאות עוד לא מחובר.
