# Bot QA Audit — 2026-05-27

Scope: `bot/ExpenseBot_FIXED.gs` (15,056 lines, build `2026-05-26-objectives-mini`)
plus the 14 Node test suites under `bot/test_*.js`.

## Summary

The bot's core expense path (Hebrew amount + free-text + classify + write) is
in solid shape: parser is well-tested, multi-business routing has a unit suite,
budget regex correctly protects "תקציב אוכל 2000" from being misread as ₪2,
account deletion is double-step, and bot-loop / pending-state-hijack guards
exist. The dispatcher correctly orders budget, objective, goal, then expense
parser, so structured commands win against the amount-extractor. However the
bot ships two active LIES about reminders ("PR הבא"), one row-delete path
with NO confirmation, the bare "יעד חדש" / multi-step flow has no state
machine to receive numeric replies, and several core commands the brief calls
out ("מנוי", project profitability "עסקה ...") are unimplemented as
top-level commands. Tests have wide breadth but zero coverage on budget regex,
support escalation, destructive confirmation, or multi-business N>9.
**Readiness: 6.5 / 10** — safe enough for the existing owner + small alpha
cohort, NOT ready for the 1000-user push without fixing the lies and the
no-confirm delete.

## Critical findings (ship-blockers)

1. **`bot/ExpenseBot_FIXED.gs:9378`** — Active lie. After "יעד חדש חודש
   לחסוך 1000" the bot replies `אזכיר אותך מספר פעמים בשבוע ... (תזכורות
   יידלקו ב-PR הבא).` The cron job for those reminders does not exist on
   the Apps Script side — the user is told "I will remind you" and won't be.
2. **`bot/ExpenseBot_FIXED.gs:9479`** — Same lie, different phrasing:
   `התראות יעדים יופעלו ב-PR הבא`. The "יעדים כבוי" command pretends to
   mute alerts that are never sent.
3. **`bot/ExpenseBot_FIXED.gs:9548`** — `התראות יישלחו אוטומטית ב-50%,
   80% ו-100% (נדלק ב-PR-2).` Same — promised, not wired.
4. **`bot/ExpenseBot_FIXED.gs:9384-9393`** — Bare "יעד חדש" sends a text
   prompt "ענה במספר 1/2/3/4" but there is no `cache.put('pendingObjective:..')`
   anywhere. The next message from the user falls through to the expense
   parser — typing "1" becomes a ₪1 expense, not a horizon selection. The
   one-shot path (`יעד חדש חודש לחסוך 1000`) works; the menu path is dead.
5. **`bot/ExpenseBot_FIXED.gs:1328`** `_handleDeleteRowCommand_` — `מחק` /
   `בטל` / `undo` deletes the user's last row instantly with no
   confirmation step. The regex is strict, but if a user types "מחק" by
   accident (or because a different command failed and they're spamming
   abort), a real expense is gone.
6. **`bot/ExpenseBot_FIXED.gs:7020`** — `מחק הזמנה` deletes the last
   business order with no confirmation.

## High findings

- **`bot/ExpenseBot_FIXED.gs:2440 parseBusinessOrder_`** — Designed for art
  orders ("עסק 880 לקוח X גודל 50-70 קנבס ..."), not for the brief's
  project-P&L shape "עסקה <client> הכנסה <n> עובדים <m>". A required flow
  is unimplemented.
- **No top-level `מנוי` handler** — brief lists this as a command. Only
  `ניקיון מנויים` (subscription cleanup) and the upgrade copy at `:7204`
  ("שדרוג"/"pro") exist.
- **No top-level `פתח עסק חדש בשם X` handler** — multi-business creation
  is implicit: typing "עסק 2 כספלה - 52 hostinger" creates business #2
  with name "כספלה". A first-time user who types `פתח עסק חדש בשם הקפה`
  gets routed to the expense parser.
- **Budget regex is untested** — `:7122` is correct today but unguarded.
  Add one assertion that "תקציב אוכל 2000" → set(אוכל, 2000) and one that
  "תקציב 2000 אוכל" doesn't false-positive.
- **Income coverage thin** — only "משכורת 12000" tested. "הכנסה מלקוח"
  not in the keyword list (`:195` has "תשלום מלקוח" but not the bare
  word). Likely classified as default → expense.
- **Test coverage on `_handleDeleteRowCommand_`** — zero. Add a regression
  that "מחק לילד" (legitimate expense for kid) does NOT delete, and that
  "מחק" alone does (or, after the fix, prompts for confirm).

## Medium findings

- **`bot/ExpenseBot_FIXED.gs:14422` SHORT_TRIGGERS** — Escalation list

  fast.
- **Build version `2026-05-26-objectives-mini`** is date-stamped (good)
  but the day before audit; confirm a deploy actually pushed.
- **`bot/ExpenseBot_FIXED.gs:6488`** — Budget empty state says "אחרי
  שתעבור 80% מהתקציב, אשלח לך התראה". The cron `api/cron/budget-check.js`

  lie.
- **Copy at `:9311`** "מזל טוב!" — single emoji is fine; consider a per-
  horizon message variant when a user marks "השגתי יעד" for a yearly goal
  versus a monthly goal.
