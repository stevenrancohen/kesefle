# Google Sheets QA Audit — 2026-05-27

**Scope:** sheet provisioning, tenant isolation, dashboards (personal + company + extended), formula health, recovery procedures, multi-business routing.
**Method:** static review of `lib/sheet-writer.js`, `lib/categories.js`, all `api/sheet/*`, `api/whatsapp/webhook.js`, `bot/ExpenseBot_FIXED.gs` write paths, `bot/personal_sheet_fix.gs`, `bot/test_isolation.js`, `docs/SHEET_FORMULAS.md`.

## Summary

Canonical write path (`api/sheet/append.js` -> `lib/sheet-writer.js`) is solid: tenant isolation via phone -> userSub -> sheet:{userSub} with leak-guard mismatch check (append.js:126-132), multi-writer anomaly detector with KV alerting (append.js:212-260), strong `bot/test_isolation.js` coverage, formula-injection sanitization (sheet-writer.js:1039), and a documented recovery flow. Biggest exposure: a **schema-incompatible legacy write path in `api/whatsapp/webhook.js`** that, if invoked, writes rows where dashboards cannot find them. **Sheets health: 7/10.**

## Critical

**`api/whatsapp/webhook.js:392-404` writes a 9-column row whose layout disagrees with the canonical template.**
- Webhook row: `[isoNow, amount, currency, type, category, subcategory, raw, source, messageId]` (A=date, B=amount, C=currency, D=type, E=category).
- Canonical `buildExpenseRow` (sheet-writer.js:1066-1093): `[isoDate, "YYYY-MM" month, amount, category, subcategory, raw, source, isExpense, vatDeductible]` (B=month, C=amount, D=cat, E=subcategory).
- Every personal/extended dashboard SUMIFS keys on `B` (month) + `E` (subcategory); company adds `D="עסק"`. Webhook rows place currency in C and "expense"/"income" in D — **dashboards return 0 for those rows.** Mitigation: webhook may be effectively unused (bot bridges via /api/sheet/append) but the handler is still wired in `vercel.json`. Recommend deleting the write block or routing through `appendRowToUserSheet`.

## High

1. **Picker -> categories drift only enforced by review.** `_CATEGORY_PICKER_GROUPS_` (bot:5756-5776) lists 17 top-level labels: 16 match `EXPENSE_GROUPS[*].label`, "עסק" is correctly the col-D top-level (not a subcategory). No drift today, but a single typo on either side is silent. Add a node test asserting every picker label is in `ALL_EXPENSE_TOPLEVEL ∪ {"עסק"}`.

2. **Company-dashboard SUMIFS may miss live data.** `COMPANY_EXPENSE_ROWS` (sheet-writer.js:98-105) is 4 buckets with wildcard criteria. Bot subcategories include `מחזור` (revenue, line 107 of FIXED.gs). If a tenant logs revenue via `/api/sheet/append` as `קטגוריה=עסק, תת=מחזור`, the company dashboard misses it (revenue is sourced from the `הזמנות` tab only — sheet-writer.js:443-453). Either route revenue through `הזמנות` or add a "📥 הכנסות עסקיות" row.

3. **Tab-name strings hardcoded in 33+ places.** Consumers redeclare `TX_TAB = 'תנועות'` (delete-last.js:28, relabel-row.js:32, fix-company-dashboard.js:32, import/bank-csv.js:49) and use inline strings (account.js:196, summary.js:86, tax-report.js:130). `lib/sheet-writer.js:1032` exports them — consumers should import. A rename of `תנועות` today would silently break half the codebase.

4. **Hardcoded section-total row numbers.** Personal-dashboard `=B28+B34+B39+B50+B58` (sheet-writer.js:288, 291) and company `=SUM(B8:B11)` (line 497) bake row positions into formulas. Documented (lines 214-246), but inserting a row above shifts numbers. Extended-dashboard tab uses computed `incomeSectionTotalRows` — the right pattern.

## Medium

- **Doc drift:** `docs/SHEET_FORMULAS.md:7-8` says `תנועות` is 8 columns; canonical is 9 (col I = ניכוי מע״מ, added 2026-05-24, sheet-writer.js:43).
- **Multi-business is owner-only.** Comments at ExpenseBot_FIXED.gs:10681 say "Tenants via /api/sheet/append still use their single provisioned sheet." Tenants typing "עסק 2 ..." write to the single תנועות tab with `קטגוריה="עסק 2"` — no auto-tab. Document this user-facing.
- **Reprovision orphans the old sheet** (admin/reprovision-user-sheet.js:267). Acceptable as an admin escape hatch.

## Recovery procedures verified

Present:
- `_isBrokenDashFormula_`, `CLEAN_BROKEN_FORMULAS`, `CLEAN_BROKEN_FORMULAS_ALL_YEARS`, `RECOMPUTE_COMPANY_DASHBOARD` (personal_sheet_fix.gs:457, 479, 968, 564).
- `docs/RECOVER_DASHBOARD_V2_RUNBOOK.md` (dry-run + apply, backup-tab step).
- `api/sheet/fix-company-dashboard.js` (single-user surgical repair).
- `api/admin/reprovision-user-sheet.js` (whole-sheet rebuild, requireAdmin).
- `api/sheet/delete-last.js`, `delete-rows.js` (targeted row deletion with metadata read first).
- `bot/test_isolation.js` (5-scenario isolation incl. SHEET_OWNER_PHONE-unset regression).

Missing:
- **No personal-dashboard recompute** analogous to `RECOMPUTE_COMPANY_DASHBOARD` for `מאזן אישי`. If a user's personal SUMIFS tangle, only path is reprovision.
- **No automated test of picker -> EXPENSE_GROUPS alignment** or of `COMPANY_EXPENSE_ROWS` -> classifier subcategory coverage.
- **No assertion that `appendRowToUserSheet` is the only canonical-format write path.** A grep-test that every `values/.*:append` call routes through `buildExpenseRow` would lock the schema.

## Top recommendations

1. Remove or refactor `api/whatsapp/webhook.js:340-440` write block — schema mismatch is a latent CRITICAL.
2. Centralize tab-name constants: import from `lib/sheet-writer.js` instead of redeclaring (~20 files).
3. Add a node test asserting every picker label is a known top-level and every `COMPANY_EXPENSE_ROWS` criterion matches a real classifier subcategory.
