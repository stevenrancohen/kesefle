# Kesefle — Keyword Universe (Hebrew-first) → Page Map

Each cluster targets one **primary page**. Use the cluster's terms naturally in
that page's `<title>`, meta description, H1/H2, body, and schema. Long-tail
question phrases → blog posts / FAQ. **Don't keyword-stuff** — one primary
intent per page, supporting terms woven in naturally.

> Brand note: people search **"כספלה"** (no geresh) far more than **"כסף'לה"**.
> Both, plus "כסףלה" and "kesefle", are now in the homepage schema. Keep using
> "כספלה" in copy + alt text so Google ties the brand to the domain.

## Location keywords — ⚠️ handle carefully
The terms ("מעקב הוצאות תל אביב", "ניהול תקציב חיפה", …) exist, but Kesefle is a
**nationwide SaaS, not a local business**. Ten near-duplicate city pages =
"doorway pages" (a Google spam violation that gets sites demoted). Instead:
one page **"מעקב הוצאות בישראל"** that naturally mentions it works everywhere
(תל אביב, ירושלים, חיפה, באר שבע, …) in a single, genuine paragraph. See
`SEO_STRATEGY.md`.
