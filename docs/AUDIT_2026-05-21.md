## Deferred — verified NOT a live bug (do NOT "fix" blindly)

- **Cookie-auth name/type mismatch.** `requireAuth` reads a `kfl_session` cookie
  and verifies it as a Google ID token; the session cookie is actually named
  `kefle_session` and is an HS256 JWT. **But nothing sets `kfl_session`, and the
  live frontend authenticates with `Authorization: Bearer <idToken>`** (verified
  in account.html:756 + dashboard.html:1323), which the Bearer path verifies
  correctly. So there is **no live 401** — the cookie branch is dead code.
  Action: optional cleanup later; do not rewrite auth without a real repro.

- **`api/whatsapp/webhook.js` write path.** Incomplete "Phase 2" scaffolding
  (stub `parseMessage`, no canonical-sheet guard, uses the tokenless `phone:`
  record). It is **fail-closed today** (can't write — no token) and is **not the
  production webhook** (the Apps Script bot is). No live leak. **Action: do NOT
  wire this endpoint as the live Meta webhook until it uses the
  `resolveTenantWriteRecord` pattern (token from `user:{sub}`, sheet from
  canonical `sheet:{sub}`, abort on mismatch).**

## Deferred — real but needs human/visual judgment

- **Color contrast (WCAG 1.4.3).** `text-ink-300` (~2:1) / `text-ink-400`
  (~2.9:1) used as body/caption text on light backgrounds fails 4.5:1
  (index/about/family/help/pricing). Fix is `text-ink-500`+ for informational
  text — but it's a broad change mixed with decorative em-dashes and dark-mode
  pairings, so it needs per-instance visual review, not a blind sed.

- **Dead code:** `lib/middleware.js` exports (`withSecurityHeaders`,
  `withValidation`, …) are never imported. `lib/billing.js:103` hardcodes the
  owner phone as a fallback (matches the bot's OWNER_PHONE; not a secret).
