# Author a Kesefle keyword pack

Packs are the human-editable source for classifier vocabulary. Each `bot/keywords/packs/*.json` file is ONE `{category,subcategory,isIncome,keywords[]}` bucket that later gets merged into the embedded `const CATEGORY_MAP = [...]` array in `bot/ExpenseBot_FIXED.gs` by the index builder ([[kesefle-keyword-index-build]]). The matcher is length-sorted (longest keyword wins) and stable (among equal lengths the earlier CATEGORY_MAP row wins). You are authoring vocab, not touching the live bot file.

## Pack shape (copy this skeleton)
A pack is exactly one bucket object. Hebrew is RAW here (escaping is the builder's job):
```json
{
  "category": "עסק",
  "subcategory": "עלות שיווק",
  "isIncome": false,
  "keywords": ["שיווק", "פרסום", "קמפיין", "פייסבוק", "facebook ads",
               "instagram ads", "google ads", "טאבולה", "ppc"]
}
```
Income bucket flips `isIncome` and targets a revenue subcategory:
```json
{ "category": "עסק", "subcategory": "מחזור", "isIncome": true,
  "keywords": ["מכירה", "תשלום לקוח", "החזר מעמ", "invoice paid"] }
```

## Steps
1. Pick the exact target bucket FIRST. The `category`+`subcategory` pair must already exist as a row in `bot/ExpenseBot_FIXED.gs` `CATEGORY_MAP` (grep it) AND map to a real dashboard row — e.g. business opex `{"category":"עסק","subcategory":"עלות שיווק"}`, income `{"category":"עסק","subcategory":"מחזור","isIncome":true}`. If no bucket fits, STOP — a new bucket is a CATEGORY_MAP + dashboard-row change, not a pack.
2. Name the file for the bucket, ASCII slug only: `bot/keywords/packs/biz-marketing.json`, `income-vat-refund.json`. One bucket per file; never mix two subcategories.
3. Shape: `{ "category": "...", "subcategory": "...", "isIncome": false, "keywords": ["...", "..."] }`. Set `isIncome:true` ONLY for revenue buckets (it flips col H sign); the builder carries it into CATEGORY_MAP verbatim.
4. Keywords are literal substrings the matcher seeks inside normalized user text. Mirror the legacy vocab style in `bot/KESEFLE_KEYWORDS_v2.gs` / `bot/KESEFLE_KEYWORDS_EXTRA_v3.gs` (brands, slang, Hebrew + English, common typos). Hebrew goes RAW in the pack JSON (e.g. `"שיווק"`, `"פייסבוק"`); the `\u`-escaping happens only at index-build time.
5. Bucket-fit rule: every keyword must be UNambiguous for THIS bucket. A bare 2-3 char token (`מעמ`, `גז`, `בר`, `פז`) belongs nowhere — it sign-flips or mis-routes (real case: bare `מעמ` booked a VAT refund as a company expense, PR #199/#200). Prefer the disambiguated phrase: `"החזר מעמ"` in the income pack, not `"מעמ"`.
6. Dedup INSIDE the pack: no repeated strings, no a-string-that-is-a-substring-of-another-in-the-same-pack (the longer one already covers it). Cross-pack collisions are resolved by the builder; do not pre-resolve them here, but avoid obvious twins.
7. Add a `tests/golden_set.js` anchor for each genuinely new routing claim (label = expected `category`/`subcategory`) so accuracy is gated. Granularity-only additions that sum to the same dashboard row do not need an anchor.

## Verification
- `node -e "JSON.parse(require('fs').readFileSync('bot/keywords/packs/<file>.json','utf8'))"` — pack must be valid JSON.
- For each notable keyword, dry-replay it through the REAL classifier (read-only, no live writes): `node bot/bot-replay.js "350 <keyword>"` and confirm predicted subcategory + `isIncome` match the bucket. (bot-replay loads CATEGORY_MAP via balanced-brace extraction of `bot/ExpenseBot_FIXED.gs`.)
- `node tests/golden_set.js` — accuracy must not drop below the file's threshold; new anchors pass.

## Common pitfalls
- Adding a bare ambiguous token to force a match — that breaks the never-corrupt floor; let it fall to ASK instead.
- A keyword whose bucket is income but `isIncome` left false (or vice-versa) — that is a silent sign-flip on the dashboard; double-check revenue packs.
- Authoring `category`/`subcategory` text that has no matching CATEGORY_MAP row or dashboard row — the bot will write a label the dashboard SUMIFS never sums.
- Editing `bot/ExpenseBot_FIXED.gs` directly here — packs are the input; CATEGORY_MAP is regenerated by [[kesefle-keyword-index-build]], then shipped via manual paste ([[bot-deploy-paste]]). Agents never push main.
